# **Специальные типы данных в Go**  
---

### **На этой странице кратко описан каждый тип, для подробной информации читайте**:  
- [[Указатели]]  
- [[Интерфейсы]]  
- [[Каналы]]  

---

## **1. Указатели (`*T`, `&x`)**  
**Ключевые свойства**:  
- Хранят **адрес памяти** переменной, а не значение  
- Нулевое значение: `nil`  
- Используются для:  
  - Модификации переменных в функциях  
  - Работы с большими структурами без копирования  
  - Создания связных структур данных  

**Пример**:  
```go
x := 10
ptr := &x  // Получение адреса
*ptr = 20  // Изменение значения через указатель
```

---

## **2. Интерфейсы (`interface{}`)**  
**Ключевые свойства**:  
- Определяют **набор методов** (контракт)  
- Пустой интерфейс `interface{}` может хранить любой тип  
- Используются для:  
  - Полиморфизма  
  - Обработки неизвестных типов данных  

**Пример**:  
```go
var val interface{} = "hello"  // Пустой интерфейс
s := val.(string)             // Type assertion
```

---

## **3. Каналы (`chan T`)**  
**Ключевые свойства**:  
- Примитив для **безопасной коммуникации** между горутинами  
- Типы:  
  - Буферизованные (`make(chan int, 3)`)  
  - Небуферизованные (`make(chan int)`)  
- Используются для:  
  - Синхронизации горутин  
  - Параллельной обработки данных  

**Пример**:  
```go
ch := make(chan int)
go func() { ch <- 42 }()
value := <-ch  // Получение значения
```

---

### **Сравнительная таблица**  

| Тип          | Нулевое значение | Основное назначение               | Риски                     |
|--------------|------------------|-----------------------------------|---------------------------|
| Указатели    | `nil`            | Изменение переменных, оптимизация | Висячие ссылки            |
| Интерфейсы   | `nil`            | Полиморфизм, обработка типов      | Неправильное приведение   |
| Каналы       | `nil`            | Синхронизация горутин             | Блокировки, deadlocks     |

---

### **Когда использовать?**  
1. **Указатели** — при работе с:  
   - Большими структурами  
   - Функциями, изменяющими параметры  
2. **Интерфейсы** — для:  
   - Универсальных обработчиков данных  
   - Моков в тестировании  
3. **Каналы** — в случаях:  
   - Параллельной обработки流水线 (pipelines)  
   - Событийно-ориентированного дизайна
