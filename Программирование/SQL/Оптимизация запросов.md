# Оптимизация запросов в PostgreSQL

## Оглавление

### **1. [[Анализ планов выполнения]]**
*EXPLAIN, чтение планов, статистика выполнения*
- Команды EXPLAIN и EXPLAIN ANALYZE
- Чтение и интерпретация планов запросов
- Анализ стоимости операций (cost)
- Статистика ввода/вывода (BUFFERS)
- Выявление узких мест производительности

---

### **2. [[Оптимизация JOIN операций]]**
*Nested Loop, Hash Join, Merge Join*
- Алгоритмы соединения таблиц
- Nested Loop Join для маленьких таблиц
- Hash Join для больших несортированных данных
- Merge Join для отсортированных данных
- Выбор стратегии соединения

---

### **3. [[Кэширование и материализованные представления]]**
*Кэш запросов, материализованные views, подготовленные выражения*
- Кэширование планов запросов
- Материализованные представления
- Подготовленные выражения (prepared statements)
- Кэширование на уровне приложения
- Стратегии инвалидации кэша

---

### **4. [[Масштабирование и репликация]]**
*Репликация, шардинг, партиционирование*
- Репликация чтения (read replicas)
- Горизонтальное шардирование
- Партиционирование таблиц
- Федеративные базы данных
- Балансировка нагрузки

---

### **5. [[Оптимизация индексов]]**
*Выбор индексов, покрывающие индексы, частичные индексы*
- Анализ использования индексов
- Покрывающие индексы (covering indexes)
- Частичные индексы (partial indexes)
- Многоколоночные индексы
- Мониторинг и обслуживание индексов

---

### **6. [[Статистика и мониторинг]]**
*Сбор статистики, мониторинг производительности*
- Статистика выполнения запросов (pg_stat_statements)
- Мониторинг медленных запросов
- Анализ использования ресурсов
- Настройка автовакуума (autovacuum)
- Профилирование нагрузки
