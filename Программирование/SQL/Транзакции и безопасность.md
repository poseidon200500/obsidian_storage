# Транзакции и безопасность в PostgreSQL

## Оглавление

### **1. [[Транзакции ACID]]**
*Атомарность, согласованность, изолированность, долговечность*
- Команды BEGIN, COMMIT, ROLLBACK
- Свойства ACID и их реализация в PostgreSQL
- WAL (Write-Ahead Log) и гарантии долговечности
- Точки сохранения (SAVEPOINT)
- Обработка ошибок в транзакциях

---

### **2. [[Уровни изоляции транзакций]]**
*Read Committed, Repeatable Read, Serializable*
- Стандартные уровни изоляции ANSI SQL
- Read Committed - поведение по умолчанию
- Repeatable Read и snapshot isolation
- Serializable - строгая сериализуемость
- Аномалии параллельного доступа

---

### **3. [[Система блокировок PostgreSQL]]**
*Табличные, строковые, advisory locks*
- Типы блокировок и их конфликты
- Блокировки таблиц (ACCESS SHARE, EXCLUSIVE, etc.)
- Блокировки строк (FOR UPDATE, FOR SHARE)
- Advisory locks - прикладные блокировки
- Мониторинг и разрешение взаимоблокировок

---

### **4. [[Управление доступом и RLS]]**
*Роли, привилегии, Row Level Security*
- Система ролей и наследование привилегий
- GRANT/REVOKE - управление правами доступа
- Row Level Security (RLS) - безопасность на уровне строк
- Политики доступа для SELECT, INSERT, UPDATE, DELETE
- Column-level privileges

---

### **5. [[Защита от SQL-инъекций]]**
*Параметризованные запросы, валидация, экранирование*
- Принципы параметризации запросов
- Подготовленные выражения (prepared statements)
- Валидация входных данных на уровне БД
- Безопасное динамическое построение запросов
- Аудит и мониторинг подозрительной активности

---

### **6. [[Мониторинг и аудит безопасности]]**
*Логирование, отслеживание, анализ уязвимостей*
- Настройка логирования безопасности
- Мониторинг активных транзакций и блокировок
- Аудит привилегий и прав доступа
- Обновления безопасности и патчинг
- Best practices для безопасной конфигурации
