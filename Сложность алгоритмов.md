# **Сложность алгоритмов**  
*Теоретическая оценка эффективности алгоритмов по времени выполнения и потребляемой памяти. Фундаментальная концепция, необходимая для анализа и сравнения различных подходов к решению задач.*

#### **1. [[Асимптотические нотации]]**  
*Математические инструменты для описания поведения алгоритма при росте объема входных данных.*  
- Big O нотация (верхняя граница)
- Big Ω нотация (нижняя граница)  
- Big Θ нотация (точная оценка)
- Практическое применение нотаций
- Частые ошибки в определении сложности

#### **2. [[Анализ временной сложности]]**  
*Методы оценки времени выполнения алгоритма в зависимости от размера входных данных.*  
- Константное время O(1)
- Логарифмическое время O(log n)
- Линейное время O(n)
- Квадратичное время O(n²)
- Факториальное время O(n!)

#### **3. [[Анализ пространственной сложности]]**  
*Оценка объема памяти, требуемой алгоритмом для работы.*

- Память под входные данные
- Рекурсивный стек вызовов
- Trade-off время/память
- Оптимизация использования памяти

*Метод анализа алгоритмов, где редкие дорогостоящие операции распределяются по многим дешевым операциям.*
